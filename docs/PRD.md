# 세무사 전문 서비스 앱

## 1. 프로젝트 개요

### 1.1 프로젝트명
텍스챗(TaxChat)

### 1.2 프로젝트 목적
사용자가 세금 신고 결제를 진행하고, 신고에 필요한 서류를 업로드·관리하며, SMS 알림을 발송할 수 있는 풀스택 웹 애플리케이션을 구축한다.

### 1.3 주요 기능(스코프)
(1) 세금 신고 결제 기능
- 사용자가 세금 신고 서비스를 결제
- 결제 내역 저장 및 조회 가능
(2)	세금 신고 관리 화면
- 결제된 세금 신고 목록 조회
- 상태값 변경(예: 결제 완료 → 신고 진행 중 → 신고 완료)
(3)	서류 첨부 / 관리 기능
- 세금 신고에 필요한 서류 업로드
- 파일 목록 조회, 다운로드, 삭제 기능
(4)	SMS 발송 및 관리
- 사용자를 대상으로 한 SMS 메시지 작성
- 발송 이력 조회

## 2. 기술 스택
프런트엔드: React, HTML, CSS, JavaScript
백엔드: Node.js, Express
데이터베이스: PostgreSQL
기타: 사용자 인증 기능 포함, 결제 기능 포함, 파일 업로드 기능 포함, SMS 발송 기능 포함

## 3. 개발 방식 및 구조

### 3.1 개발 구조
- 프런트엔드와 백엔드 완전 분리 개발
- RESTful API 방식으로 통신

### 3.2 기본 원칙
- 기본적인 웹 기술만 사용 (불필요한 프레임워크 지양)
- API 명확한 스펙 정의 필요
- 모든 기능은 로그인 기반으로 동작
- 파일 업로드는 서버 또는 S3 등 외부 스토리지 사용 가능

## 4. 요구사항 요약

### 4.1 기능 요구사항
- 결제 화면: 결제 요청/완료 처리, 결제 내역 저장
- 결제 내역 관리: 필터/검색, 상태 관리
- 파일 첨부: 업로드/다운로드/삭제
- SMS: 작성/저장/발송/이력 관리
- 인증: 회원가입, 로그인, 권한관리(필요 시)

### 4.2 비기능 요구사항
- 반응형 UI
- 기본적인 로깅
- 에러 처리(프런트/백 공통 정책)

## 5. 메인 화면 상세 설계

### 5.0 공통 UI 원칙
- **모바일 전용 설계**: 모든 사용자 화면은 모바일 해상도 기준으로만 제작한다.
- **PC/태블릿 레이아웃**: PC나 태블릿 환경에서는 모바일 비율 화면을 중앙에 배치하고, 나머지 영역은 데코레이션으로 채운다.
  - 모바일 화면: 전체 화면 사용 (높이: 100vh/100dvh)
  - PC/태블릿 화면: 모바일 앱을 중앙에 배치
    - 화면 비율: **20:9 (가로:세로)** - `aspect-ratio: 9 / 20`
    - 너비 계산: `calc(100vh * 9 / 20)`
    - 최소 너비: 430px (Galaxy S25 Ultra 기준)
    - 높이: 100vh/100dvh (전체 화면 높이와 일치)
  - 배경 데코레이션: 그라데이션 배경, 부유 애니메이션 원형 요소, 패턴 오버레이
  - 모바일 앱 스타일: 둥근 모서리, 그림자 효과, 파란색 테두리 글로우
  - 반응형으로 확장하지 않음, 모바일 고정 UI 적용
  - 헤더는 모바일 앱 컨테이너 내부에 포함되어 화면을 벗어나지 않도록 처리

### 5.1 화면 개요
메인 화면은 사용자가 앱에 접속했을 때 가장 먼저 보이는 화면으로, 주요 기능으로의 진입점을 제공한다. 헤더, 메인 배너, 주요 기능 버튼으로 구성된다.

### 5.2 화면 구성 요소

#### 5.2.1 헤더 영역 (Header)
**위치**: 화면 최상단
**레이아웃**: 좌우 정렬

**구성 요소**:
- **좌측**: 로고 이미지 표시
- **우측**: 
  - 알림 아이콘 (종 모양 버튼)
  - 메뉴 아이콘 (햄버거 버튼)

**스타일 요구사항**:
- 헤더는 상단 고정(fixed) 처리
- 스크롤 시에도 항상 화면 상단에 표시
- z-index 값 설정하여 다른 콘텐츠 위에 표시

#### 5.2.2 메인 배너 영역 (Main Banner Slider)
**위치**: 헤더 바로 아래
**레이아웃**: 전체 너비, 슬라이더 형태

**기능 요구사항**:
- 좌/우 슬라이드 가능한 메인 배너

**인디케이터**:
- 배너 하단에 현재 배너 번호 / 전체 배너 수 형태로 인디케이터 표시
- 예: 1 / 3

#### 5.2.3 기능 버튼 영역 (Action Buttons)
**위치**: 메인 배너 아래
**레이아웃**: 모바일 UI 기준으로 세로 배치

**버튼 구성**:

1. **결제하기 버튼**
   - 세금 신고 결제 화면으로 이동
   - 모바일 전체 너비 기준으로 크게 배치

2. **서류첨부하기 버튼**
   - 서류 업로드 화면으로 이동
   - 모바일 전체 너비 기준으로 크게 배치

**스타일 요구사항**:
- 버튼은 모바일 전체 너비 기준으로 크게 배치
- 터치 영역 충분히 확보
- 터치 시 명확한 피드백 제공

#### 5.2.4 푸터 콘텐츠 (Footer Content)
**위치**: 메인 콘텐츠 하단
**레이아웃**: 중앙 정렬, 세로 배치

**구성 요소**:
- "사용문의" 텍스트
- "카카오채널" 링크 (밑줄 표시, 클릭 가능)
- "byray" 브랜드명

**스타일 요구사항**:
- 텍스트 크기: 작은 크기 (12px)
- 색상: 회색 계열 (#64748b)
- 세로 간격: 적절한 여백 유지 (0.5rem)
- 별도의 푸터 영역 없이 메인 콘텐츠 하단에 직접 출력

#### 5.2.5 카카오톡 상담 버튼 (KakaoTalk Consultation Button)
**위치**: 화면 우측 하단 고정
**레이아웃**: 플로팅 버튼 형태

**구성 요소**:
- 카카오톡 아이콘 (k_logo.png 이미지)
- "카카오톡 상담" 텍스트

**스타일 요구사항**:
- 위치: 화면 우측 하단 고정 (`position: fixed`)
  - 하단 여백: 7rem (`bottom: 7rem`)
  - 우측 여백: 1rem (`right: 1rem`)
- 배경색: 어두운 회색 (#333333)
- 텍스트 색상: 흰색 (#ffffff)
- 텍스트 크기: 1.2rem
- 폰트 굵기: 500
- 패딩:
  - 상단: 3px
  - 하단: 3px
  - 좌측: 15px
  - 우측: 24px
- 아이콘 크기: 56px x 56px
- 아이콘 배경색: 없음 (투명)
- 둥근 모서리: 50px (완전히 둥근 형태)
- 그림자: 0 4px 12px rgba(0, 0, 0, 0.15)
- z-index: 999 (다른 요소 위에 표시)
- 호버 효과: 배경색 변경 (#444444), 약간 위로 이동
- 클릭 시: 카카오톡 채널로 이동

**PC/태블릿 스타일**:
- 위치: 모바일 앱 컨테이너 내부에 절대 위치 (`position: absolute`)
  - 하단 여백: 7rem (`bottom: 7rem`)
  - 우측 여백: 1rem (`right: 1rem`)

### 5.3 사용자 인터랙션

- 터치 디바이스에서 터치 영역 충분히 확보
- 로딩 상태 표시 (필요 시)

### 5.4 접근성

- 충분한 색상 대비
- 명확한 터치 영역 표시

### 5.5 PC/태블릿 레이아웃 상세

#### 5.5.1 레이아웃 구조
- **모바일 화면 (< 768px)**: 전체 화면 사용, 배경 데코레이션 숨김
- **태블릿/PC 화면 (≥ 768px)**: 모바일 앱을 중앙에 배치
  - 화면 비율: **20:9 (가로:세로)** - `aspect-ratio: 9 / 20`
  - 너비: `calc(100vh * 9 / 20)`
  - 최소 너비: 430px
  - 높이: 100vh/100dvh
- **데스크톱 화면 (≥ 1024px)**: 더 큰 여백과 데코레이션 요소

#### 5.5.2 모바일 앱 컨테이너
- **위치**: 화면 중앙에 배치
- **화면 비율**: **20:9 (가로:세로)**
  - PC/태블릿에서 `aspect-ratio: 9 / 20` 적용
  - 너비: `calc(100vh * 9 / 20)`
  - 최소 너비: 430px (Galaxy S25 Ultra 기준)
  - 높이: 100vh/100dvh (전체 화면 높이와 일치)
- **스타일**:
  - 둥근 모서리: 24px (PC/태블릿에서만 적용)
  - 그림자 효과: 깊이감 있는 그림자
  - 파란색 테두리 글로우: 브랜드 색상 강조
- **헤더 처리**:
  - 헤더는 모바일 앱 컨테이너 내부에 포함
  - PC/태블릿에서 `position: absolute`로 처리하여 컨테이너를 벗어나지 않도록 함
  - `max-width: 100%` 및 `box-sizing: border-box` 적용

#### 5.5.3 배경 데코레이션
- **그라데이션 배경**: 파란색 계열 그라데이션 (연한 파란색 → 밝은 파란색)
- **부유 애니메이션 원형 요소**: 
  - 3개의 원형 요소가 부드럽게 움직이는 애니메이션
  - 반투명 파란색 그라데이션 적용
  - 각기 다른 크기와 위치
- **패턴 오버레이**: 방사형 그라데이션 패턴으로 깊이감 추가

#### 5.5.4 반응형 동작
- 모바일: 전체 화면 사용, 배경 데코레이션 비표시
- 태블릿/PC: 모바일 앱 중앙 배치, 배경 데코레이션 표시
- 모든 화면 크기에서 모바일 앱의 UI는 동일하게 유지

### 5.6 마이 페이지 (My Page)

#### 5.6.1 화면 개요
사용자의 개인 정보 및 서비스 이용 관련 메뉴를 제공하는 화면이다. 메인 화면의 우측 상단 메뉴 버튼을 클릭하면 이동한다.

#### 5.6.2 화면 구성 요소

**헤더 영역**:
- **좌측**: 뒤로가기 버튼 (메인 화면으로 이동)
- **중앙**: "마이 페이지" 제목

**회원 정보 영역**:
- **회원명 표시**:
  - 회원 유형에 따라 다른 필드명 표시:
    - **"비사업자"**인 경우: "성명" 라벨과 함께 회원명 표시
    - **"개인 사업자"**인 경우: "사업장명" 라벨과 함께 사업장명 표시
    - **"법인 사업자"**인 경우: "사업장명" 라벨과 함께 사업장명 표시
- **회원 유형 라벨**:
  - 회원명/사업장명 옆에 현재 회원 유형을 라벨로 표시
  - 라벨 값: "비사업자", "개인 사업자", "법인 사업자" 중 하나
  - 둥근 모서리의 버튼 형태로 표시
- **회원 유형 변경 버튼**:
  - 라벨 옆에 배치
  - 버튼 텍스트: "회원 유형 변경"
  - 클릭 시 회원 유형 변경 화면으로 이동 또는 모달 표시

**메뉴 항목**:
1. **서류 보관함**: 업로드된 서류 목록 조회 페이지로 이동
2. **서비스 이용내역**: 서비스 이용 내역 조회 페이지로 이동
3. **로그아웃**: 로그아웃 처리

**회원탈퇴**:
- 화면 하단에 회색 텍스트로 표시
- 클릭 시 회원탈퇴 확인 후 처리

#### 5.6.3 기능 요구사항

**회원 정보 표시**:
- 회원 유형에 따라 적절한 필드명과 값 표시
- 비사업자: "성명" + 회원명
- 개인 사업자/법인 사업자: "사업장명" + 사업장명
- 현재 회원 유형을 라벨로 표시

**회원 유형 변경**:
- "회원 유형 변경" 버튼 클릭 시 회원 유형 선택 화면으로 이동
- 회원 유형 변경 가능 (비사업자 ↔ 개인 사업자 ↔ 법인 사업자)

### 5.6.4 회원 유형 선택 화면 (Member Type Selection)

#### 5.6.4.1 화면 개요
사용자가 가입한 회원 유형을 선택하거나 새로운 회원 유형을 추가할 수 있는 화면이다. 마이 페이지의 "회원 유형 변경" 버튼을 클릭하면 이동한다.

#### 5.6.4.2 화면 구성 요소

**헤더 영역**:
- **좌측**: 뒤로가기 버튼 (마이 페이지로 이동)
- **중앙**: "회원 유형 선택" 제목

**회원 유형 카드 섹션**:
각 가입한 회원 유형마다 카드 형태로 구성되며, 다음 정보를 포함한다:

1. **선택 라디오 버튼**:
   - 카드 왼쪽에 위치
   - 현재 선택된 회원 유형은 보라색 채워진 원으로 표시
   - 선택되지 않은 회원 유형은 빈 회색 원으로 표시
   - 클릭 시 해당 회원 유형 선택

2. **회원 정보**:
   - **회원명/사업장명**: 큰 굵은 글씨로 표시 (예: "최민용", "by ray")
   - **회원 유형 라벨**: 회원명/사업장명 아래에 둥근 모서리의 버튼 형태로 표시
     - 라벨 값: "비사업자", "개인 사업자", "법인 사업자" 중 하나
     - 얇은 검은색 테두리, 흰색 배경

3. **더보기 버튼**:
   - 카드 오른쪽 위에 세로 점 3개 아이콘으로 표시
   - 클릭 시 모달이 열림

**더보기 모달**:
- 더보기 버튼 클릭 시 해당 회원 유형 카드 근처에 표시
- 모달 내용:
  - **상세보기 버튼**: 해당 회원 유형의 상세 정보 페이지로 이동
  - **삭제 버튼**: 해당 회원 유형 삭제 (확인 후 처리)
- 모달 외부 클릭 시 닫힘

**회원 유형 추가 버튼**:
- 회원 유형 카드들 아래에 위치
- 버튼 형태: 검은색 배경, 흰색 텍스트
- 버튼 텍스트: "+ 회원 유형"
- 왼쪽에 흰색 플러스(+) 아이콘 표시
- 클릭 시 새로운 회원 유형 추가 화면으로 이동 또는 모달 표시

**선택 완료 버튼**:
- 화면 하단에 위치
- 버튼 형태: 보라색 배경, 흰색 텍스트, 둥근 모서리
- 버튼 텍스트: "선택 완료"
- 전체 너비로 배치
- 클릭 시 선택한 회원 유형으로 변경하고 마이 페이지로 이동

#### 5.6.4.3 기능 요구사항

**회원 유형 선택**:
- 라디오 버튼 클릭 시 해당 회원 유형 선택
- 한 번에 하나의 회원 유형만 선택 가능
- 선택된 회원 유형은 보라색 라디오 버튼으로만 강조 표시
- 카드 배경색은 변경되지 않음 (항상 흰색 배경 유지)

**더보기 기능**:
- 더보기 버튼 클릭 시 모달 표시
- 모달에는 "상세보기"와 "삭제" 버튼 포함
- 상세보기: 해당 회원 유형의 상세 정보 페이지로 이동
- 삭제: 확인 후 해당 회원 유형 삭제 처리

**회원 유형 추가**:
- "+ 회원 유형" 버튼 클릭 시 새로운 회원 유형 추가 화면으로 이동
- 회원 유형 추가 후 목록에 자동 반영

**선택 완료**:
- "선택 완료" 버튼 클릭 시:
  - 선택한 회원 유형으로 변경 처리
  - 마이 페이지로 이동
  - 변경된 회원 유형이 마이 페이지에 반영됨

#### 5.6.4.4 데이터 구조

**회원 유형 카드 데이터**:
- 회원 ID
- 회원명/사업장명
- 회원 유형 ("비사업자", "개인 사업자", "법인 사업자")
- 선택 여부 (현재 활성화된 회원 유형인지)

#### 5.6.4.5 스타일 요구사항
- 회원 유형 카드는 흰색 배경, 얇은 검은색 테두리로 구분
- 선택된 카드는 보라색 라디오 버튼으로 강조
- 회원명/사업장명은 큰 굵은 글씨로 표시
- 회원 유형 라벨은 둥근 모서리 버튼 형태
- 더보기 버튼은 세로 점 3개 아이콘
- "+ 회원 유형" 버튼은 검은색 배경, 흰색 텍스트
- "선택 완료" 버튼은 보라색 배경, 흰색 텍스트, 전체 너비
- 카드 간 적절한 간격 유지

### 5.6.5 회원 유형 추가 화면 (Add Member Type)

#### 5.6.5.1 화면 개요
사용자가 새로운 회원 유형을 추가할 수 있는 화면이다. 회원 유형 선택 화면의 "+ 회원 유형" 버튼을 클릭하면 이동한다. 이미 가입한 회원 유형은 제외하고, 가입하지 않은 회원 유형만 표시된다.

#### 5.6.5.2 화면 구성 요소

**헤더 영역**:
- **좌측**: 뒤로가기 버튼 (회원 유형 선택 페이지로 이동)
- **중앙**: "회원 유형 추가" 제목

**회원 유형 카드 섹션**:
가입하지 않은 회원 유형만 카드 형태로 표시되며, 각 카드는 다음 정보를 포함한다:

1. **회원 유형 제목**:
   - 큰 굵은 글씨로 표시
   - 값: "비사업자", "개인 사업자", "법인 사업자" 중 하나

2. **회원 유형 설명**:
   - 제목 아래에 표시
   - 각 회원 유형별 세금 납부 의무에 대한 설명
   - **비사업자**: "사업을 하지 않는 일반 개인은 근로소득세, 종합소득세, 양도세 등 소득·자산 거래에서 발생하는 세금을 납부합니다. 부동산·자동차를 사면 취득세, 보유하면 재산세가 발생합니다."
   - **개인 사업자**: "개인사업자는 매출이 생기면 부가세와 종합소득세를 기본적으로 신고·납부해야 합니다. 직원·프리랜서에게 비용을 지급하면 원천징수세가 생길 수 있으며, 지방소득세도 함께 납부해야 합니다."
   - **법인 사업자**: "법인은 사업에서 발생한 이익에 대해 법인세를 납부하고, 매출에 따른 부가세 신고도 필수입니다. 직원 급여·배당 지급 시 원천징수 의무가 있으며 지방소득세도 함께 납부합니다."

**카드 선택 상태**:
- **선택하지 않은 상태**: 흰색 배경, 검은색 테두리, 검은색 텍스트
- **선택한 상태**: 검은색 배경, 흰색 텍스트
- 카드 클릭 시 선택/해제 토글

**선택 완료 버튼**:
- 화면 하단에 위치
- 버튼 형태: 보라색 배경, 흰색 텍스트, 둥근 모서리
- 버튼 텍스트: "선택 완료"
- 전체 너비로 배치
- 클릭 시 선택한 회원 유형을 추가하고 회원 유형 선택 페이지로 이동

#### 5.6.5.3 기능 요구사항

**회원 유형 필터링**:
- 이미 가입한 회원 유형은 목록에서 제외
- 가입하지 않은 회원 유형만 표시
- 예: 비사업자와 개인 사업자를 이미 가입했다면, 법인 사업자만 표시

**회원 유형 선택**:
- 카드 클릭 시 선택/해제 토글
- 한 번에 하나의 회원 유형만 선택 가능
- 선택된 카드는 검은색 배경, 흰색 텍스트로 표시

**회원 유형 추가**:
- "선택 완료" 버튼 클릭 시:
  - 선택한 회원 유형에 대한 추가 폼 페이지로 이동
  - 회원 유형별로 다른 폼이 표시됨:
    - **비사업자**: 성명, 성별, 주민등록번호, 주소지 입력 폼
    - **개인 사업자**: 사업장명, 대표자명, 사업자번호, 업종, 업태, 사업장 주소, 사업 시작일 입력 폼
    - **법인 사업자**: 법인명, 대표자명, 사업자번호, 업종, 업태, 사업장 주소, 사업 시작일 입력 폼
  - 폼 작성 완료 후 회원 유형 선택 페이지로 이동
  - 새로 추가된 회원 유형이 목록에 표시됨

#### 5.6.5.4 데이터 구조

**회원 유형 데이터**:
- 회원 유형 코드 ("비사업자", "개인 사업자", "법인 사업자")
- 회원 유형 제목
- 회원 유형 설명
- 가입 여부 (이미 가입한 유형인지)

#### 5.6.5.5 스타일 요구사항
- 회원 유형 카드는 흰색 배경, 얇은 검은색 테두리로 시작
- 선택된 카드는 검은색 배경, 흰색 텍스트로 변경
- 회원 유형 제목은 큰 굵은 글씨로 표시
- 회원 유형 설명은 작은 글씨로 표시
- "선택 완료" 버튼은 보라색 배경, 흰색 텍스트, 전체 너비
- 카드 간 적절한 간격 유지

**기타 기능**:
- 각 메뉴 항목 클릭 시 해당 페이지로 이동
- 로그아웃 시 확인 후 메인 화면으로 이동
- 회원탈퇴 시 확인 후 처리

### 5.7 서비스 이용내역 화면 (Service Usage History)

#### 5.7.1 화면 개요
사용자가 결제한 서비스의 이용 내역을 조회하고, 각 이용 건의 상태를 확인하며, 첨부 서류 업로드 및 수정, 납부 확인서 다운로드 등의 기능을 제공하는 화면이다. 마이 페이지의 "서비스 이용내역" 메뉴를 클릭하면 이동한다.

#### 5.7.2 화면 구성 요소

**헤더 영역**:
- **좌측**: 뒤로가기 버튼 (마이 페이지로 이동)
- **중앙**: "서비스 이용 내역" 제목

**이용 내역 카드 섹션**:
각 이용 건마다 카드 형태로 구성되며, 다음 정보를 포함한다:

1. **서비스 정보**:
   - 서비스명 (예: "인건비 신고")
   - 결제일시 (예: "25.08.16. 10:35")
   - 서비스 유형 (예: "상시근로자 (4대 보험신고 필요)")

2. **첨부 서류 상태**:
   - "<첨부 서류 상태>" 라벨
   - 각 첨부 서류별 상태 표시:
     - 서류 보관함에 저장된 서류인 경우: "첨부 서류명 (첨부 완료)"
     - 서류 보관함에 저장되지 않은 서류인 경우: "첨부 서류명 (첨부 필요)"

3. **처리 상태 표시**:
   - 3개의 상태 버튼이 가로로 배치됨:
     - "결제 완료"
     - "신고 진행 중"
     - "신고 완료"
   - 현재 상태에 해당하는 버튼만 검은색 배경으로 강조 표시
   - 나머지 버튼은 흰색 배경, 검은색 테두리로 표시

4. **액션 버튼**:
   - 상태 값에 따라 다른 버튼이 표시됨:
     - **"첨부 필요" 상태일 때**: "첨부 서류 업로드하기" 버튼 (보라색)
     - **"첨부 완료" 상태일 때**: "첨부 서류 수정하기" 버튼 (보라색)
     - **"신고 완료" 상태일 때**: "납부 확인서 다운 받기" 버튼 (보라색)

5. **안내 문구**:
   - 액션 버튼 아래 빨간색 텍스트로 안내 문구 표시:
     - "첨부 서류 업로드하기" 버튼 아래: "첨부 서류 업로드가 완료되어 상태가 '첨부 서류 수정하기' 상태로 변경되어야만 신고를 진행할 수 있습니다."
     - "첨부 서류 수정하기" 버튼 아래: "신고 진행 중에도 이미 업로드된 첨부 서류는 계속 수정할 수 있습니다."
     - "납부 확인서 다운 받기" 버튼 아래: "신고가 완료되면 납부확인서를 다운로드할 수 있습니다."

#### 5.7.3 상태 값 관리

**처리 상태 값**:
- 각 이용 건은 다음 3가지 상태 중 1개를 가짐:
  1. **"결제 완료"**: 결제가 완료된 상태
  2. **"신고 진행 중"**: 첨부 서류 업로드가 완료되어 신고가 진행 중인 상태
  3. **"신고 완료"**: 신고가 완료된 상태

**첨부 서류 상태 값**:
- 각 첨부 서류는 다음 2가지 상태 중 1개를 가짐:
  1. **"첨부 완료"**: 서류 보관함에 저장된 서류인 경우
  2. **"첨부 필요"**: 서류 보관함에 저장되지 않은 서류인 경우

**상태 값과 버튼 표시 규칙**:
- 첨부 서류가 모두 "첨부 필요" 상태인 경우:
  - 처리 상태: "결제 완료"
  - 표시 버튼: "첨부 서류 업로드하기"
  - 안내 문구: "첨부 서류 업로드가 완료되어 상태가 '첨부 서류 수정하기' 상태로 변경되어야만 신고를 진행할 수 있습니다."

- 첨부 서류가 모두 "첨부 완료" 상태인 경우:
  - 처리 상태: "신고 진행 중"
  - 표시 버튼: "첨부 서류 수정하기"
  - 안내 문구: "신고 진행 중에도 이미 업로드된 첨부 서류는 계속 수정할 수 있습니다."

- 처리 상태가 "신고 완료"인 경우:
  - 표시 버튼: "납부 확인서 다운 받기"
  - 안내 문구: "신고가 완료되면 납부확인서를 다운로드할 수 있습니다."

#### 5.7.4 기능 요구사항

**첨부 서류 업로드하기**:
- 버튼 클릭 시 첨부 서류 업로드 화면으로 이동
- 모든 필수 첨부 서류 업로드 완료 시 처리 상태가 "신고 진행 중"으로 자동 변경
- 버튼이 "첨부 서류 수정하기"로 변경됨

**첨부 서류 수정하기**:
- 버튼 클릭 시 첨부 서류 수정 화면으로 이동
- 이미 업로드된 서류를 수정하거나 새로운 서류로 교체 가능
- 신고 진행 중에도 계속 수정 가능

**납부 확인서 다운 받기**:
- 버튼 클릭 시 납부 확인서 파일 다운로드
- 신고 완료 상태에서만 사용 가능

**데이터 조회**:
- 사용자의 모든 결제 내역을 최신순으로 표시
- 각 이용 건의 현재 상태를 실시간으로 반영

#### 5.7.5 스타일 요구사항
- 카드 형태의 레이아웃으로 각 이용 건을 구분
- 상태 버튼은 가로로 배치하며, 현재 상태는 검은색 배경으로 강조
- 액션 버튼은 보라색 배경으로 표시
- 안내 문구는 빨간색 텍스트로 표시하여 주의를 환기
- 카드 간 적절한 간격 유지

### 5.8 서류 보관함 화면 (Document Storage Box)

#### 5.8.1 화면 개요
사용자가 업로드한 서류를 조회하고 관리할 수 있는 화면이다. 마이 페이지의 "서류 보관함" 메뉴를 클릭하면 이동한다.

#### 5.8.2 화면 구성 요소

**헤더 영역**:
- **좌측**: 뒤로가기 버튼 (마이 페이지로 이동)
- **중앙**: "서류 보관함" 제목

**서류 첨부 버튼**:
- **위치**: 헤더 바로 아래, 화면 상단
- **버튼 텍스트**: "서류 첨부"
- **스타일**: 검은색 배경, 흰색 텍스트, 전체 너비
- **기능**: 클릭 시 서류 첨부 페이지로 이동

**첨부된 서류 영역**:
- **제목**: "첨부된 서류 ({개수})" 형태로 표시
  - 예: "첨부된 서류 (2)"
- **정렬 버튼**: 제목 오른쪽에 배치
  - 버튼 형태: 둥근 모서리, 연한 회색 테두리
  - 버튼 텍스트: 현재 선택된 정렬 옵션 표시
  - 클릭 시 정렬 옵션 드롭다운 또는 선택 메뉴 표시

**정렬 옵션**:
1. "첨부일시 최신순"
2. "첨부일시 오래된 순"
3. "가나다 순"
4. "가나다 역순"

**서류 카드**:
각 서류는 카드 형태로 표시되며, 다음 정보를 포함한다:

1. **서류 정보**:
   - 서류종류 (예: "사업자등록증", "주민등록등본")
   - 서류종류는 굵은 검은색 텍스트로 표시

2. **첨부일시**:
   - 형식: "첨부일시 : YY.MM.DD hh:mm:ss"
   - 예: "첨부일시 : 25.11.28 13:30:45"
   - 서류종류 아래에 표시

3. **액션 버튼**:
   - 카드 하단에 가로로 배치
   - **상세 보기 버튼** (보라색):
     - 버튼 텍스트: "상세 보기"
     - 클릭 시 해당 서류 상세보기 페이지로 이동
   - **삭제 버튼** (빨간색):
     - 버튼 텍스트: "삭제"
     - 클릭 시 삭제 확인 다이얼로그 표시

#### 5.8.3 기능 요구사항

**서류 첨부**:
- "서류 첨부" 버튼 클릭 시 서류 첨부 페이지로 이동
- 서류 업로드 후 서류 보관함에 자동 반영

**정렬 기능**:
- 정렬 버튼 클릭 시 정렬 옵션 선택 가능
- 선택된 정렬 옵션에 따라 서류 목록 재정렬
- 기본 정렬: "첨부일시 최신순"

**서류 조회**:
- 사용자가 업로드한 모든 서류를 카드 형태로 표시
- 서류가 없는 경우 안내 메시지 표시

**상세 보기**:
- "상세 보기" 버튼 클릭 시 해당 서류의 상세 정보 페이지로 이동
- 상세 정보 페이지에서 서류 내용 확인 및 다운로드 가능

**서류 삭제**:
- "삭제" 버튼 클릭 시 확인 다이얼로그 표시
- 확인 다이얼로그 메시지: "해당 서류를 삭제하시겠습니까?"
- 확인 다이얼로그 버튼:
  - "삭제" 버튼: 서류 삭제 처리
  - "닫기" 버튼: 다이얼로그 닫기 (삭제 취소)
- 삭제 완료 후 서류 목록에서 제거

#### 5.8.4 데이터 형식

**첨부일시 형식**:
- 형식: YY.MM.DD hh:mm:ss
- 예시: "25.11.28 13:30:45"
- 년도: 2자리 (YY)
- 월: 2자리 (MM)
- 일: 2자리 (DD)
- 시간: 24시간 형식 (hh:mm:ss)

#### 5.8.5 스타일 요구사항
- 서류 카드는 흰색 배경, 연한 회색 테두리로 구분
- 서류종류는 굵은 검은색 텍스트로 강조
- 첨부일시는 회색 텍스트로 표시
- 상세 보기 버튼은 보라색 배경, 흰색 텍스트
- 삭제 버튼은 빨간색 배경, 흰색 텍스트
- 카드 간 적절한 간격 유지
- 정렬 버튼은 연한 회색 테두리, 둥근 모서리

### 5.9 서류 첨부 화면 (Document Attachment)

#### 5.9.1 화면 개요
사용자가 서류를 선택하고 파일을 업로드하여 서류 보관함에 저장하는 화면이다. 서류 보관함 페이지의 "서류 첨부" 버튼을 클릭하면 모달 형태로 표시된다.

#### 5.9.2 화면 구성 요소

**헤더 영역**:
- **좌측**: "서류 첨부" 제목
- **우측**: 닫기 버튼 (X 아이콘)
  - 클릭 시 모달 닫기 및 서류 보관함 페이지로 돌아가기

**서류 선택 영역**:
- **서류 선택 콤보 박스**:
  - 위치: 헤더 바로 아래
  - 형태: 둥근 모서리의 입력 필드
  - 플레이스홀더: "첨부할 서류를 선택해주세요"
  - 우측에 드롭다운 화살표 아이콘 표시
  - 클릭 시 서류 목록 드롭다운 표시

**서류 목록**:
- 관리자 페이지의 첨부 서류관리 화면에서 등록한 첨부 서류 내역이 리스트로 출력됨
- 예시: "가족관계증명서", "사업자등록증", "주민등록등본", "기타" 등
- 서류 선택 시 콤보 박스에 선택한 서류명 표시

**서류 검색 기능**:
- 서류 선택 콤보 박스에서 텍스트 입력 시 검색 가능
- 입력한 글자와 일치하는 서류가 필터링되어 표시됨
- 실시간 검색 (입력과 동시에 필터링)

**서류 찾기 버튼**:
- 위치: 서류 선택 콤보 박스 바로 아래
- 초기 상태: 비활성화 (회색 배경)
- 활성화 조건: 서류 선택 시 활성화 (검은색 배경)
- 버튼 텍스트: "서류 찾기"
- 기능: 클릭 시 기기의 파일 찾기 기능 활성화

**선택한 파일 표시 영역**:
- 파일 선택 후 아래 영역에 표시됨
- **파일명 표시**:
  - 선택한 파일명 표시 (예: "가족서류.pdf")
  - 우측에 삭제 버튼 (X 아이콘)
- **파일 설명 입력란**:
  - 위치: 파일명 표시 아래
  - 형태: 텍스트 영역
  - 플레이스홀더: "첨부한 서류에 대해 설명이 필요하다면 간략히 설명하세요"
  - 필수 입력 사항 아님

**파일 삭제 기능**:
- 파일명 우측의 X 버튼 클릭 시:
  - 선택한 파일 리스트 제거
  - 파일 설명 입력란 제거
  - "서류 찾기" 버튼 다시 활성화

**서류 첨부 완료 버튼**:
- 위치: 화면 하단
- 형태: 보라색 배경, 흰색 텍스트, 둥근 모서리
- 버튼 텍스트: "서류 첨부 완료"
- 초기 상태: 비활성화
- 활성화 조건: 필수 입력 사항 입력 시 활성화
  - 필수 입력 사항: 서류 선택, 파일 선택
  - 선택 입력 사항: 파일 설명 (입력하지 않아도 됨)

#### 5.9.3 기능 요구사항

**서류 선택**:
- 콤보 박스 클릭 시 관리자 페이지에서 등록한 서류 목록 표시
- 목록에서 서류 선택 가능
- 선택한 서류명이 콤보 박스에 표시됨
- 서류 선택 시 "서류 찾기" 버튼 활성화

**서류 검색**:
- 콤보 박스에 텍스트 입력 시 실시간 검색
- 입력한 글자와 일치하는 서류만 필터링되어 표시
- 검색 결과에서 서류 선택 가능

**파일 선택**:
- "서류 찾기" 버튼 클릭 시 기기의 파일 선택 기능 활성화
- 파일 선택 후:
  - 선택한 파일명 표시
  - 파일 설명 입력란 표시
  - "서류 찾기" 버튼 비활성화

**파일 삭제**:
- 파일명 우측 X 버튼 클릭 시:
  - 선택한 파일 제거
  - 파일 설명 입력란 제거
  - "서류 찾기" 버튼 다시 활성화

**서류 첨부 완료**:
- 필수 입력 사항 (서류 선택, 파일 선택) 입력 시 버튼 활성화
- 버튼 클릭 시:
  - 서류 보관함에 서류 저장
  - 서류 보관함 페이지로 이동 또는 모달 닫기
  - 업로드 완료 메시지 표시 (선택 사항)

#### 5.9.4 데이터 연동

**서류 목록 데이터**:
- 관리자 페이지의 첨부 서류관리 화면에서 등록한 서류 목록을 가져옴
- 서류명, 서류 설명 등이 포함됨
- 사용 가능한 서류만 표시 (사용 여부가 "사용"인 서류)

**파일 업로드**:
- 선택한 파일을 서버에 업로드
- 업로드된 파일 정보를 서류 보관함에 저장
- 서류명, 첨부일시, 파일 경로 등 저장

#### 5.9.5 스타일 요구사항
- 모달 형태로 화면 중앙에 표시
- 흰색 배경, 검은색 테두리
- 서류 선택 콤보 박스: 둥근 모서리, 드롭다운 화살표 아이콘
- 서류 찾기 버튼: 비활성화 시 회색, 활성화 시 검은색 배경
- 선택한 파일 표시: 파일명과 X 버튼이 함께 표시
- 파일 설명 입력란: 텍스트 영역 형태
- 서류 첨부 완료 버튼: 보라색 배경, 비활성화 시 회색
- 모든 요소 간 적절한 간격 유지

## 6. 관리자 화면 상세 설계

### 6.1 화면 개요
관리자 화면은 시스템 관리자가 고객, 상품, 결제, 서류, SMS 등을 관리할 수 있는 통합 관리 대시보드이다. 모든 관리자 화면은 공통 헤더와 네비게이션 바를 공유하며, 각 화면은 특정 관리 기능에 집중한다.

### 6.2 공통 구성 요소

#### 6.2.1 헤더 영역 (Header)
**위치**: 화면 최상단
**레이아웃**: 좌우 정렬

**구성 요소**:
- **좌측**: 로고 이미지
  - 클릭 시 관리자 메인 화면으로 이동
  - 이미지 크기: 적절한 비율 유지
  
- **우측**: 로그아웃 버튼
  - 버튼 텍스트: "로그아웃"
  - 클릭 시 로그아웃 처리 및 로그인 화면으로 이동

**스타일 요구사항**:
- 헤더는 모든 관리자 화면에서 고정 위치로 유지
- 배경색: 흰색
- 하단에 구분선(border) 추가

#### 6.2.2 네비게이션 바 (Navigation Bar)
**위치**: 헤더 바로 아래
**레이아웃**: 가로 정렬

**메뉴 항목** (좌측부터 순서대로):
1. **고객 관리** (Customer Management)
2. **상품 관리** (Product Management)
3. **주문 결제 관리** (Order Payment Management)
4. **첨부 서류 관리** (Attached Document Management)
5. **SMS 관리** (SMS Management)
6. **환경 설정** (Environment Settings)

**기능 요구사항**:
- 각 메뉴 버튼 클릭 시 해당 관리 화면으로 이동
- 현재 활성화된 메뉴는 보라색 배경으로 표시
- 비활성 메뉴는 흰색 배경, 검은색 테두리로 표시

**스타일 요구사항**:
- 버튼 형태: 직사각형
- 활성 메뉴: 보라색 배경, 흰색 텍스트
- 비활성 메뉴: 흰색 배경, 검은색 텍스트, 검은색 테두리
- 버튼 간 적절한 간격 유지

#### 6.2.3 공통 컨트롤 영역 (Common Controls)
**위치**: 네비게이션 바 아래, 테이블 위
**레이아웃**: 좌우 정렬

**좌측 영역**:
- **총 개수 표시**: "총 {총개수}" 텍스트
- **선택 개수 표시**: "선택 {선택한 개수} 건" 텍스트
- **전체 선택 버튼**: "전체 선택"
  - 클릭 시 테이블의 모든 항목 선택/해제 토글

**우측 영역**:
- **페이지당 리스트 수 드롭다운**: "페이지당 리스트 수"
  - 옵션: 10, 20, 30, 50, 100 (기본값: 50)
  - 선택 시 테이블 항목 수 변경 및 페이지네이션 재계산

**스타일 요구사항**:
- 좌우 영역 간 적절한 간격 유지
- 텍스트와 버튼의 가독성 확보

#### 6.2.4 페이지네이션 (Pagination)
**위치**: 테이블 하단, 중앙 정렬

**구성 요소**:
- **이전 버튼**: "이전"
  - 첫 페이지에서는 비활성화
  - 클릭 시 이전 페이지로 이동
- **페이지 번호**: 숫자 버튼들
  - 현재 페이지는 검은색 배경, 흰색 텍스트로 강조
  - 페이지가 많을 경우 생략 표시("...") 사용
  - 예시: "1", "2", "3", "...", "35"
- **다음 버튼**: "다음"
  - 마지막 페이지에서는 비활성화
  - 클릭 시 다음 페이지로 이동

**기능 요구사항**:
- 페이지 번호 클릭 시 해당 페이지로 이동
- 페이지 변경 시 테이블 데이터 갱신

### 6.3 고객 관리 화면

#### 6.3.1 화면 개요
고객(회원) 정보를 조회, 등록, 수정, 삭제할 수 있는 화면이다. 상단 메뉴에서 "고객 관리"를 선택하면 접근할 수 있다.

#### 6.3.2 주요 기능
- 회원 목록 조회 (페이지네이션 지원)
- 회원 등록
- 회원 상세 정보 조회

- 수정 모드에서 "[취소]" 버튼 클릭 시 원래 값으로 복원되고 읽기 모드로 돌아감

**상품 카테고리 등록 모달**:
- "[상품 카테고리 등록]" 버튼 클릭 시 상품 카테고리 등록 모달이 출력됨

  - 제목: "상품 카테고리 등록" (상단 중앙)
  - **[창닫기] 버튼** (오른쪽 상단)
    - 클릭 시 모달이 닫힘
- **입력 필드**:
  - **상품 카테고리명** (필수):
    - 단일 라인 텍스트 입력 필드
    - placeholder: "상품 카테고리명 입력"
    - 입력한 값은 상품 카테고리 관리 페이지의 "상품 카테고리명" 컬럼에 출력됨
  - **상품 카테고리 간략 설명** (선택):
    - textarea 입력 필드
    - placeholder: "상품 카테고리 간략 설명 입력"
    - 입력한 값은 상품 카테고리 관리 페이지의 "상품 간략 카테고리 설명" 컬럼에 출력됨
  - **상품 카테고리 상세 설명** (선택):
    - textarea 입력 필드
    - placeholder: "상품 카테고리 상세 설명 입력"
    - 입력한 값은 상품 카테고리 관리 페이지의 "상품 상세 카테고리 설명" 컬럼에 출력됨
- **액션 버튼**:
  - **[취소] 버튼**: 모달 닫기
  - **[등록] 버튼**: 
    - 초기 상태: 비활성화 (회색 배경)
    - 상품 카테고리명 입력 시 활성화 (검은색 배경)
    - 활성화된 상태에서 클릭 시 상품 카테고리 저장 및 등록
- **상품 카테고리 코드 자동 생성**:
  - 저장 시 상품 카테고리 코드가 자동으로 발급됨
  - 생성 규칙: "cate" + 두 자리 순번 숫자 + 소문자 알파벳 임의의 두 글자
  - 예시: "cate01ab", "cate02cd", "cate03ef" 등
  - 유일한 상품 카테고리 코드가 생성됨 (중복 방지)

### 6.5 상품 관리 화면

#### 6.5.1 화면 개요
상품 정보를 조회, 등록, 수정, 삭제할 수 있는 화면이다. 상단 메뉴에서 "상품 관리"를 선택하면 접근할 수 있다.

#### 6.5.2 주요 기능
- 상품 목록 조회 (페이지네이션 지원)
- 상품 등록
- 상품 상세 정보 조회
- 상품 수정
- 상품 삭제 (DB는 유지, 출력만 제외)
- 진열/비진열 상태 변경
- 검색 및 필터링 (등록일시, 검색어, 진열 여부)

#### 6.5.3 화면 구성 요소

**검색 영역** (네비게이션 바 아래):
- **등록일시 검색**:
  - 빠른 선택 버튼: "오늘", "1주일", "1개월", "3개월", "6개월", "1년", "전체"
  - 날짜 범위 입력 필드: 시작일, 종료일 (YYYY-MM-DD 형식)
  
- **검색어 입력**:
  - 검색 기준 선택 드롭다운: "상품 카테고리명", "상품명", "상품코드"
  - 검색어 입력 필드: placeholder "검색어 입력"
  
- **진열 여부 필터**:
  - 체크박스 형태: "[진열]", "[비진열]"
  - 여러 개 선택 가능
  
- **액션 버튼**:
  - "[검색]" 버튼 (회색 배경)
  - "[초기화]" 버튼 (흰색 배경, 검은색 테두리)

**공통 컨트롤 영역** (검색 영역 아래, 테이블 위):
- **좌측 영역**:
  - **액션 버튼**:
    - "[상품 카테고리 관리]" 버튼 (빨간색)
    - "[상품 등록]" 버튼 (보라색)
  - "총 {총개수}" 텍스트
  - "선택 {선택한 개수} 건" 텍스트
  - "[전체 선택]" 버튼
  
- **우측 영역**:
  - **정렬 순서 선택 드롭다운**:
    - 옵션: "등록일시순", "등록일시 역순"
    - 기본값: "등록일시순"
  - **페이지당 리스트 수 드롭다운**:
    - 옵션: 10, 20, 30, 50, 100 (기본값: 50)

**데이터 테이블**:
- **테이블 컬럼**:
  1. 체크박스 (행 선택)
  2. 상품 카테고리명 (Product Category Name)
     - 상품 카테고리 관리 페이지에서 생성된 값 출력
  3. 상품명 (Product Name)
  4. 상품코드 (Product Code)
  5. 상품 가격 (Product Price)
     - 원화 형식으로 표시 (예: "50,000원")
  6. 진열 (Display Status)
     - "[진열]" 또는 "[비진열]" 버튼 형태로 출력
     - 기본값: "비진열" (상품이 사용자 페이지에 출력되지 않음)
     - "[비진열]" 버튼 클릭 시 "[진열]" 버튼으로 변경되며 상품이 출력됨
     - "[진열]" 버튼 클릭 시 "[비진열]" 버튼으로 변경되며 상품이 출력되지 않음
  7. 기능 (Function)
     - 상세 보기 버튼
     - 삭제 버튼
  8. 등록일 (Registration Date)
     - 상품 등록일시를 YYYY-MM-DD hh:mm:ss 형태로 출력

**페이지네이션**:
- 테이블 하단에 위치
- 이전/다음 버튼 및 페이지 번호 표시

#### 6.5.4 기능 요구사항

**검색 기능**:
- 등록일시 빠른 선택 버튼 클릭 시 해당 기간으로 날짜 범위 자동 설정
- 검색 기준 선택 후 검색어 입력하여 검색
- 진열 여부 체크박스로 필터링 (여러 개 선택 가능)
- "[검색]" 버튼 클릭 시 필터링된 결과 표시
- "[초기화]" 버튼 클릭 시 모든 검색 조건 초기화

**정렬 기능**:
- 정렬 순서 드롭다운에서 "등록일시순" 또는 "등록일시 역순" 선택
- 기본값: "등록일시순"
- 정렬 순서 변경 시 테이블 데이터 재정렬

**진열 상태 관리**:
- 진열 컬럼에 "[진열]" 또는 "[비진열]" 버튼 표시
- 기본값: "비진열" (상품이 사용자 페이지에 출력되지 않음)
- "[비진열]" 버튼 클릭 시 "[진열]" 버튼으로 변경되며 상품이 사용자 페이지에 출력됨
- "[진열]" 버튼 클릭 시 "[비진열]" 버튼으로 변경되며 상품이 사용자 페이지에서 제거됨

**삭제 기능**:
- "[삭제]" 버튼 클릭 시 해당 상품 정보가 출력되지 않음
- DB에는 데이터가 남아있으며, 출력만 하지 않음

**상품 등록 모달**:
- "[상품 등록]" 버튼 클릭 시 상품 등록 모달이 출력됨
- **모달 헤더**:
  - 제목: "상품 등록" (상단 중앙)
  - **[창닫기] 버튼** (오른쪽 상단)
    - 클릭 시 모달이 닫힘
- **상품 카테고리 선택**:
  - 드롭다운 형태의 입력 필드
  - 클릭 시 상품 카테고리 관리 페이지에 등록된 상품 카테고리 리스트가 출력되어 선택 가능
  - 선택한 상품 카테고리는 상품 관리 페이지의 "상품 카테고리명" 컬럼에 출력됨
- **상품 정보 입력 영역 (반복 가능)**:
  - "상품 + 순번" 형태로 표시 (예: "상품 01", "상품 02")
  - 각 상품 섹션 오른쪽에 "[상품 +]" 버튼과 "[상품 삭제]" 버튼이 위치
  - "[상품 +]" 버튼 클릭 시 현재 상품 아래에 다음 순번의 상품 정보 입력 영역이 추가됨
  - "[상품 삭제]" 버튼 클릭 시 해당 상품 정보 입력 영역이 삭제됨
- **각 상품 정보 입력 필드**:
  - **상품 이용 가능자** (필수):
    - 체크박스 형태: "비사업자", "개인 사업자", "법인 사업자"
    - 하나 이상 선택 필수
    - 선택한 회원 유형만 해당 상품을 선택 결제할 수 있음
  - **상품명** (필수):
    - 텍스트 입력 필드
    - placeholder: "상품명 입력"
    - 입력한 값은 상품 관리 페이지의 "상품명" 컬럼에 출력됨
  - **상품 가격** (필수):
    - 텍스트 입력 필드 (숫자만 입력 가능)
    - placeholder: "숫자만 입력"
    - 입력 필드 옆에 "원" 단위 표시
    - 입력한 값은 상품 관리 페이지의 "상품 가격" 컬럼에 출력됨
  - **상품 설명** (필수):
    - textarea 입력 필드
    - placeholder: "상품 설명 입력"
  - **첨부 서류 선택** (선택):
    - 드롭다운 형태의 입력 필드
    - 클릭 시 첨부 서류 리스트가 출력되어 선택 가능
    - 선택 후 우측의 "[+]" 버튼 클릭 시 "첨부 서류 + 순번" 형태로 첨부 서류가 추가됨
    - 각 첨부 서류 우측에 "[-]" 버튼이 출력되며, 클릭 시 해당 첨부 서류가 삭제됨
- **액션 버튼**:
  - **[취소] 버튼**: 모달 닫기
  - **[등록] 버튼**: 
    - 초기 상태: 비활성화 (회색 배경)
    - 모든 필수 필드 입력 시 활성화 (검은색 배경)
    - 활성화된 상태에서 클릭 시 상품 등록 저장
- **상품 코드 자동 생성**:
  - 저장 시 각 상품에 대해 상품 코드가 자동으로 발급됨
  - 생성 규칙: "cate" + 두 자리 순번 숫자 + 소문자 알파벳 임의의 두 글자 + "_" + 두 자리 순번
  - 예시: "cate01ab_01", "cate01ab_02", "cate02cd_01" 등
  - 유일한 상품 코드가 생성됨 (중복 방지)

**상품 카테고리 관리**:
- "[상품 카테고리 관리]" 버튼 클릭 시 상품 카테고리 관리 페이지로 이동
- 상품 카테고리 관리 페이지에서 생성된 카테고리명이 상품 관리 페이지의 "상품 카테고리명" 컬럼에 출력됨

     - 주문이 없는 경우 "-" 표시
  4. 회원 (Member Name)
     - 회원명 출력
  5. 서류명 (Document Name)
     - 첨부된 서류명 출력
  6. 메모 (Memo)
     - 고객 관리 페이지의 "메모" 컬럼과 유사하게, 고객별 최신 작성한 메모가 출력됨
     - 최신 메모가 있을 경우: 메모 내용이 최대 25자까지만 표시되며, 25자 이상일 경우 말줄임표("...")를 사용하여 출력
     - 최신 메모가 없을 경우: "[메모 출력]" 버튼이 표시됨
     - 메모 내용 또는 "[메모 출력]" 버튼 클릭 시 메모 모달이 출력됨
  7. 기능 (Function)
     - 다운로드 버튼
     - 삭제 버튼

**페이지네이션**:
- 테이블 하단에 위치
- 이전/다음 버튼 및 페이지 번호 표시

#### 6.7.4 기능 요구사항

**검색 기능**:
- 첨부일시 빠른 선택 버튼 클릭 시 해당 기간으로 날짜 범위 자동 설정

**메모 기능**:
- 메모 컬럼에 최신 메모 내역이 출력됨
- 최신 메모가 있을 경우: 메모 내용이 최대 25자까지만 표시되며, 25자 이상일 경우 말줄임표("...")를 사용하여 출력
- 최신 메모가 없을 경우: "[메모 출력]" 버튼이 표시됨
- 메모 내용 또는 "[메모 출력]" 버튼 클릭 시 메모 모달이 출력됨

  - **등록일시 검색**:
    - 빠른 선택 버튼: "오늘", "1주일", "1개월", "3개월", "6개월", "1년", "전체"

  - **검색어 입력**:

  - **액션 버튼**:
    - "[검색]" 버튼 (회색 배경)

  - **우측 영역**:

    - **[첨부 서류 등록] 버튼** (보라색)

    - **정렬 순서 선택 드롭다운**:
      - 옵션: "등록일시순", "등록일시 역순"

    - **페이지당 리스트 수 드롭다운**:

- **기능 요구사항**:

  - **삭제 기능**: "[삭제]" 버튼 클릭 시 해당 서류 정보가 출력되지 않음, DB에는 데이터가 남아있으며, 출력만 하지 않음

    - 선택한 서류 카테고리는 서류 관리 페이지의 "서류 카테고리" 컬럼에 출력됨

    - placeholder: "서류명 입력"
    - 입력한 값은 서류 관리 페이지의 "서류명" 컬럼에 출력됨

  - "[수정하기]" 버튼 클릭 시 서류 등록 정보를 수정할 수 있는 상태로 변경됨

  - 드롭다운: "전체" 또는 특정 수신인 선택

  7. 기능 (Function)

- 등록된 템플릿이 없는 경우: "등록된 템플릿이 없습니다." 메시지 표시 (빨간색 텍스트)
- 등록된 템플릿이 있는 경우: 각 템플릿을 행으로 표시
  - **템플릿명**: 템플릿명이 표시되는 영역 (흰색 배경, 검은색 테두리)
  - **[미사용]/[사용] 버튼**: 
    - 기본값: "[미사용]" 버튼 (흰색 배경, 검은색 테두리)
    - "[미사용]" 버튼 클릭 시 "[사용]" 버튼으로 변경 (검은색 배경, 흰색 텍스트)
    - "[사용]" 버튼 클릭 시 "[미사용]" 버튼으로 변경
    - "[사용]" 상태인 템플릿만 SMS 전송 모달의 템플릿 선택 리스트에 출력됨
  - **[상세보기] 버튼**: 
    - 클릭 시 템플릿 등록 모달이 출력됨 (읽기 모드)
    - 모달에는 입력된 템플릿 정보가 표시됨
    - "[등록]" 버튼 대신 "[수정하기]" 버튼이 출력됨
  - **[삭제] 버튼**: 
    - 클릭 시 확인 후 해당 템플릿 삭제

**액션 버튼** (모달 하단):
- **[취소] 버튼**: 
  - 흰색 배경, 검은색 테두리, 검은색 텍스트
  - 클릭 시 모달 닫기
- **[저장] 버튼**: 
  - 검은색 배경, 흰색 텍스트
  - 클릭 시 변경사항 저장 및 모달 닫기

#### 6.8.6 템플릿 등록 모달

**모달 개요**:
SMS 템플릿 관리 모달의 "[템플릿 등록]" 버튼 클릭 시 템플릿 등록 모달이 출력된다.

**모달 헤더**:
- 제목: "템플릿 등록" (상단 중앙)
- **[창닫기] 버튼** (오른쪽 상단)
  - 클릭 시 모달이 닫힘

**입력 필드**:
- **템플릿명 입력** (필수):
  - 라벨: "템플릿명을 입력하세요."
  - 단일 라인 텍스트 입력 필드
  - placeholder: "템플릿명 입력"
  - 검은색 테두리
- **템플릿 내용 입력** (필수):
  - 라벨: "템플릿 내용을 입력하세요"
  - textarea 형태의 입력 필드
  - placeholder: "템플릿 내용 입력"
  - 검은색 테두리

**액션 버튼**:
- **[취소] 버튼**: 
  - 흰색 배경, 검은색 테두리, 검은색 텍스트
  - 클릭 시 모달 닫기
- **[등록] 버튼**: 
  - 초기 상태: 비활성화 (회색 배경, 회색 텍스트)
  - 템플릿명과 템플릿 내용 입력 시 활성화 (검은색 배경, 흰색 텍스트)
  - 활성화된 상태에서 클릭 시 템플릿 저장 및 SMS 템플릿 관리 모달에 추가

**템플릿 상세보기 및 수정**:
- SMS 템플릿 관리 모달의 "[상세보기]" 버튼 클릭 시 템플릿 등록 모달이 출력됨
- 모달에는 입력된 템플릿 정보가 표시됨 (읽기 전용, 회색 배경)
- "[등록]" 버튼 대신 "[수정하기]" 버튼이 출력됨 (검은색 배경, 흰색 텍스트)
- "[수정하기]" 버튼 클릭 시 템플릿 등록 정보를 수정할 수 있는 상태로 변경됨 (읽기 전용 해제)
- 수정 모드로 전환되면 "[수정하기]" 버튼이 "[수정완료]" 버튼으로 교체됨
- "[수정완료]" 버튼 클릭 시 수정한 템플릿 등록 정보로 저장되고 모달이 닫힘
- 수정 모드에서 "[취소]" 버튼 클릭 시 원래 값으로 복원되고 읽기 모드로 돌아감
- 읽기 모드에서 "[취소]" 버튼 클릭 시 모달이 닫힘

**기능 요구사항**:
- 템플릿명과 템플릿 내용 입력 시 등록 버튼 활성화
- 등록 버튼 클릭 시 템플릿 저장 및 SMS 템플릿 관리 모달에 추가
- 템플릿 사용 여부에 따라 SMS 전송 모달의 템플릿 선택 리스트에 표시 여부 결정
- 템플릿 수정 기능 제공
- 템플릿 삭제 기능 제공

#### 6.8.4 SMS 전송 모달

**모달 개요**:
SMS 관리 페이지의 "[SMS 전송]" 버튼 클릭 시 SMS 전송 모달이 출력된다.

**모달 헤더**:
- 제목: "SMS 전송" (상단 중앙)
- **[창닫기] 버튼** (오른쪽 상단)
  - 클릭 시 모달이 닫힘

**수신 번호 선택 영역**:
- **라디오 버튼 선택**:
  - "수신인 불러오기" (기본값, 선택됨)
  - "수신 번호 새로 작성"
- **수신인 불러오기** (기본 선택):
  - 검색 가능한 콤보박스 형태의 입력 필드
  - placeholder: "수신인 검색"
  - 입력 시 회원 목록이 필터링되어 표시됨
  - 회원 목록에는 회원명과 전화번호가 함께 표시됨
  - 예시: "홍길동 (010-1234-5678)"
  - 목록에서 선택 시 해당 회원의 전화번호가 자동으로 입력됨
- **수신 번호 새로 작성**:
  - 라디오 버튼 선택 시 전화번호 직접 입력 필드 표시
  - placeholder: "전화번호 입력"
  - 전화번호 형식: 010-1234-5678

**SMS 유형 선택**:
- 드롭다운 형태의 선택 필드
- 옵션: "템플릿", "결제링크", "상품 결제링크", "내용 작성"
- 선택한 SMS 유형에 따라 아래 영역이 동적으로 변경됨

**SMS 유형별 입력 영역**:

1. **템플릿** (SMS 유형이 "템플릿"인 경우):
   - **템플릿 선택 리스트**:
     - 드롭다운 형태의 선택 필드
     - placeholder: "선택"
     - SMS 템플릿 관리에서 등록한 템플릿 목록이 표시됨
     - 템플릿 선택 시 해당 템플릿 내용이 자동으로 입력됨

2. **결제링크** (SMS 유형이 "결제링크"인 경우):
   - **결제할 금액 입력**:
     - 숫자 입력 필드
     - placeholder: "결제 금액 입력"
     - 입력 필드 옆에 "원" 단위 표시
   - **전송할 내용 입력**:
     - textarea 형태의 입력 필드
     - placeholder: "전송 내용 입력"

3. **상품 결제링크** (SMS 유형이 "상품 결제링크"인 경우):
   - **상품 선택 리스트**:
     - 드롭다운 형태의 선택 필드
     - placeholder: "선택"
     - 상품 관리에서 등록한 상품 목록이 표시됨
     - 상품 선택 시 해당 상품의 결제 링크가 생성됨

4. **내용 작성** (SMS 유형이 "내용 작성"인 경우):
   - **전송할 내용 입력**:
     - textarea 형태의 입력 필드
     - placeholder: "전송 내용 입력"

**액션 버튼 영역** (모달 하단):
- **[SMS 미리보기] 버튼**:
  - 회색 배경, 회색 텍스트
  - 클릭 시 입력한 내용을 미리보기 형태로 표시
- **[취소] 버튼**:
  - 흰색 배경, 검은색 테두리, 검은색 텍스트
  - 클릭 시 모달 닫기
- **[등록] 버튼**:
  - 회색 배경, 회색 텍스트 (초기 비활성화 상태)
  - 필수 입력 사항 입력 시 활성화 (보라색 배경, 흰색 텍스트)
  - 활성화된 상태에서 클릭 시 SMS 발송 처리

**기능 요구사항**:
- 수신인 불러오기 선택 시 회원 검색 및 선택 기능 제공
- SMS 유형 변경 시 해당 유형에 맞는 입력 영역만 표시
- 필수 입력 사항 확인 후 등록 버튼 활성화
- SMS 미리보기 기능 제공
- 등록 버튼 클릭 시 SMS 발송 처리 및 모달 닫기

### 6.10 공통 기능 요구사항

#### 6.10.1 테이블 기능
- 모든 테이블은 페이지네이션 지원
- 행 선택 시 체크박스 활성화
- 전체 선택 버튼으로 모든 행 선택/해제 가능
- 선택된 항목 개수 실시간 표시

#### 6.10.2 데이터 조작
- 등록: 각 화면의 등록 버튼 클릭 시 등록 폼 표시
- 조회: 상세 보기 버튼 클릭 시 상세 정보 표시
- 수정: 상세 화면 또는 별도 수정 버튼을 통해 수정 가능
- 삭제: 삭제 버튼 클릭 시 확인 다이얼로그 표시 후 삭제 처리

#### 6.10.3 검색 및 필터링
- 각 화면별 필요에 따라 검색 및 필터 기능 제공
- 검색 조건은 쿼리 파라미터로 관리하여 URL 공유 가능 (선택 사항)

#### 6.10.4 권한 관리
- 관리자 권한이 있는 사용자만 접근 가능
- 권한이 없는 사용자는 접근 시 로그인 화면으로 리다이렉트

### 6.11 스타일 가이드

#### 6.11.2 버튼 스타일
- 주요 액션 버튼: 보라색 배경, 흰색 텍스트, 적절한 패딩
- 보조 액션 버튼: 흰색 배경, 검은색 텍스트, 테두리
